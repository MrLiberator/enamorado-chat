PROJECT CONTEXT

I am building a small interactive website for a Telegram Mafia community called **"Enamorado de un sueño"**.

The website is already deployed here:
https://enamorado-chat.vercel.app/

The site acts as a **community portal and information hub** for the Telegram chat.

The goal is to gradually evolve the site from a simple static rules page into an **interactive ecosystem with AI assistance, personality tests, and community knowledge tools**.

The project is primarily a **learning / personal development project** and there is no strict deadline.

---

CURRENT WEBSITE STRUCTURE

The site is static and intentionally lightweight.

Tech stack:

* HTML
* CSS
* Vanilla JavaScript
* Hosted on Vercel

Project structure:

/assets
/css/style.css
/js/scripts.js
index.html

Existing features already implemented:

1. Landing / welcome section

   * background image
   * chat name
   * community description
   * legal notice about accepting chat rules

2. Navigation bar

3. Tabs with sections:

   * Events archive
   * Chat rules
   * Game rules
   * Roles list

These sections already work and must NOT be broken.

The site must remain **simple, fast, and lightweight**.

Avoid heavy frameworks.

---

LONG-TERM PROJECT IDEA

I want to gradually build something like an **AI spirit of the chat**.

An assistant that:

* knows the rules
* understands mafia roles
* knows community events
* can comment on situations
* can generate small user dossiers based on chat analysis

The assistant will have a personality (possibly called **"Дявольонок" / "Little Devil"**).

The assistant should feel like a **character connected to the chat culture**.

---

CORE FEATURES TO BUILD

The development should happen in stages.

---

STAGE 1 — INTERACTIVE COMMUNITY FEATURES

Add fun interactive elements to the site.

1. Personality Test
   "Which admin are you most similar to?"

Requirements:

* 13 questions (symbolic number for the community)
* multiple choice answers
* score calculation
* result page

The result shows similarity to different admins.

Example:

You are most similar to **Admin A**

Similarity:
60% Admin A
25% Admin B
15% Admin C

---

2. Role Personality Test
   "Which Mafia role are you?"

Example results:

* Don
* Detective
* Doctor
* Maniac
* Civilian

Same structure:

* multiple choice questions
* scoring
* result screen

---

These tests must be **data-driven**.

Questions and results should be stored in JSON files instead of being hardcoded in JavaScript.

Example structure:

/data
admins.json
roles.json
tests.json

---

STAGE 2 — ASSISTANT WIDGET

Add a floating assistant widget.

Example UI:

💬 Assistant button in the corner.

Click opens a small chat window.

Assistant personality example:

"😈 Дявольонок — the spirit of the chat."

The assistant can:

* explain roles
* explain rules
* explain events
* suggest tests
* answer questions about the community

Example questions users may ask:

"What does the Don role do?"
"What happens if someone goes AFK?"
"What events happen on Friday?"
"Explain the rules of the game."

Initially the assistant can work with a **simple knowledge base** (JSON / text files).

Example knowledge structure:

/knowledge
roles.json
events.json
rules.json

---

STAGE 3 — LOCAL AI INTEGRATION

Later the assistant will use a **local AI model running on my PC**.

Reason:
I do not want to pay for OpenAI API.

Plan:

* run a local LLM using **Ollama**
* models like:

  * Llama
  * Mistral
  * Deepseek

The website will send requests to a **local AI server running on my computer**.

Architecture idea:

User → Website → API endpoint → Local AI server

The local AI server will:

* load knowledge files
* answer questions
* later integrate OSINT analysis

Access to the local AI server will be exposed through a **tunnel** such as:

Cloudflare Tunnel
or
ngrok

---

STAGE 4 — CHAT OSINT MODULE

I plan to build a **separate local program on my PC** that analyzes the Telegram chat.

Important:
This is NOT a Telegram bot.

It will be a **local OSINT parser program** that:

* parses chat history
* filters messages by user
* analyzes behavior patterns

The program will generate **mini dossiers** about users.

Example output:

User: player123

Activity:

* 421 messages
* most active at night

Role tendencies:

* often plays mafia roles

Notes:

* manipulative communication style

These dossiers will be stored locally (JSON or database).

When the AI assistant is active it will be able to answer questions like:

"Who is this user?"
"What happened yesterday in the chat?"
"Is this player usually mafia or civilian?"

---

ASSISTANT AVAILABILITY LOGIC

If my PC and AI server are online:

Assistant answers using the **local AI + OSINT data**.

If my PC is offline:

Assistant should respond with a fallback message like:

"Right now the chat is not visible to my eye.
My master is offline."

But the assistant can still answer basic questions using the local knowledge files.

---

FUTURE IDEA — BROWSER AI FALLBACK

Later I may add **WebLLM / Transformers.js** so that some AI functionality can run directly in the browser.

This would allow the assistant to still function partially even when the local AI server is offline.

The final goal is to turn the site into a **small interactive AI-powered portal for the Telegram Mafia community**, where the assistant acts like a living "spirit of the chat" that knows the rules, roles, events, and people.
